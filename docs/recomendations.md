Ниже представлен полный план-руководство по сбору «эталонного» датасета для дообучения модели **RWKV-7 7.2B** под задачи **1С:Предприятие**. 

Соблюдение этого плана позволит избежать деградации модели и «JSON-галлюцинаций».

---

# Стратегия подготовки датасета «1C-Expert-v4»

**Целевой объем:** 500 МБ – 1 ГБ чистых данных.
**Целевая пропорция (Mixing Ratio):** 
*   **50%** — Реальный код 1С (BSL).
*   **30%** — Общий кодинг (Python, JS, SQL).
*   **20%** — Общий интеллект и русский язык (Alpaca/Saiga).

---

## 1. Блок «Мясо»: Код 1С (Твоя работа)
Поскольку готовых датасетов 1С в открытом доступе нет, тебе нужно собрать его из конфигураций (БП, УТ, ЗУП, ERP).

*   **Что собирать:** Общие модули, модули менеджеров, модули объектов.
*   **Как оформлять:** Каждый метод (процедуру/функцию) делай отдельным примером.
*   **Формат:**
    ```text
    Instruction: Напиши процедуру для [ИмяПроцедуры] в 1С.
    Response: 
    Процедура ИмяПроцедуры()
        // Код
    КонецПроцедуры
    <|endoftext|>
    ```

## 2. Блок «Мозги»: Общий Кодинг (Эталоны)
Это научит модель логике программирования, которую она перенесет на 1С.

*   **MagicCoder-Evol-Instruct-110K** (Один из лучших)
    *   *Ссылка:* [Hugging Face - ise-uiuc/MagicCoder-Evol-Instruct-110K](https://huggingface.co/datasets/ise-uiuc/MagicCoder-Evol-Instruct-110K)
    *   *Почему:* Высокое качество логики, чистые примеры.
*   **CodeAlpaca-20k**
    *   *Ссылка:* [Hugging Face - sahil2801/CodeAlpaca-20k](https://huggingface.co/datasets/sahil2801/CodeAlpaca-20k)
    *   *Зачем:* Базовые задачи по программированию.

## 3. Блок «Личность»: Русский язык (Identity)
Чтобы модель не превратилась в «генератор кода», который не может связать двух слов по-русски.

*   **Saiga Scored (Ilya Gusev)** — Золотой стандарт для русского чата.
    *   *Ссылка:* [Hugging Face - IlyaGusev/saiga_scored](https://huggingface.co/datasets/IlyaGusev/saiga_scored)
*   **Russian Alpaca**
    *   *Ссылка:* [Hugging Face - IlyaGusev/ru_turbo_alpaca](https://huggingface.co/datasets/IlyaGusev/ru_turbo_alpaca)

## 4. Технический стандарт оформления (Важно!)

Чтобы в ответах не было `{"instruction": ...}`, твой скрипт подготовки данных (перед `prepare_binidx.sh`) должен склеивать JSONL в **чистый текст** с разделителями.

**Пример функции на Python для твоего препроцессора:**

```python
def format_sample(sample):
    # Убираем возможный мусор и лишние пробелы
    inst = sample['instruction'].strip()
    resp = sample['output'].strip()
    
    # Формируем финальный текстовый блок
    # В RWKV7 World используется формат Instruction: / Response:
    return f"Instruction: {inst}\n\nResponse: {resp}\n<|endoftext|>\n"

# На выходе должен получиться файл .txt, который ты потом токенизируешь
```

## 5. Итоговый чек-лист перед запуском обучения

1.  **Объем:** Проверь размер итогового текстового файла. Если он меньше **200 МБ**, продолжай собирать код 1С.
2.  **Никаких `{}`:** Открой файл в текстовом редакторе. Если видишь кавычки и фигурные скобки, которые не относятся к коду — исправляй препроцессор.
3.  **Shuffle (Перемешивание):** Обязательно перемешай строки в датасете, чтобы модель не учила сначала 1000 примеров только на русском, а потом 1000 примеров только на 1С. Она должна видеть их вперемешку.
4.  **Token End:** Убедись, что в конце каждого примера стоит `<|endoftext|>`.
