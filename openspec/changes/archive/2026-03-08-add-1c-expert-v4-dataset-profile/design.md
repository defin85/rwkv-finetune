## Context
Нужна операционная конкретизация общей стратегии датасетов для практического релиза под 1C:Enterprise: какие источники смешиваются, в каком формате готовится train-text и какие проверки блокируют выпуск.

Основной риск: деградация качества модели из-за несбалансированного корпуса и неправильной сериализации (`JSON-like` ответы вместо чистого текстового формата для RWKV).

## Goals / Non-Goals
- Goals:
  - Зафиксировать профиль `1C-Expert-v4` как воспроизводимый контракт подготовки данных.
  - Сделать pre-release проверки детерминированными и fail-closed.
  - Обеспечить совместимость с текущим pipeline `jsonl -> binidx`.
- Non-Goals:
  - Полный редизайн всех пайплайнов тренировки.
  - Внедрение новых моделей или смена оркестратора.

## Decisions
- Decision: Профиль задаётся как явный набор ограничений (объём, mix, формат, gates), а не как "best effort" рекомендации.
  - Alternatives considered: оставить рекомендации только в markdown.
  - Rationale: требуется автоматическая валидация до запуска обучения.
- Decision: Формат релизного train-текста нормализуется в `Instruction/Response + <|endoftext|>`.
  - Alternatives considered: хранить JSONL `instruction/output` до токенизации.
  - Rationale: снижает риск JSON-галлюцинаций и соответствует ожидаемому текстовому потоку для токенизации.
- Decision: Проверки объёма и mix выполняются до `prepare_binidx` и блокируют релиз при нарушениях.
  - Alternatives considered: допускать частично валидные релизы.
  - Rationale: fail-closed безопаснее для качества модели.

## Risks / Trade-offs
- Жёсткие пороги mix могут увеличивать время curating при недостатке 1C-контента.
  - Mitigation: явный отчёт дефицита по категориям и план добора данных.
- Дополнительные gates усложняют pipeline.
  - Mitigation: минимальный набор проверок с машинно-читаемым отчётом.

## Migration Plan
1. Ввести профиль `1C-Expert-v4` и отчёт pre-release в существующий data lifecycle.
2. Добавить formatter/shuffle/gates перед вызовом `prepare_binidx`.
3. Провести smoke на тестовом релизе и обновить документацию.
