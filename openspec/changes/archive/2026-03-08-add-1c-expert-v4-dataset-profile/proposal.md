# Change: Зафиксировать операционный профиль датасета 1C-Expert-v4 для RWKV v7

## Why
В репозитории есть общая стратегия разработки датасетов, но отсутствует формальный операционный профиль для практической сборки релиза по рекомендациям из `docs/recomendations.md`.

Без такого профиля команда рискует получать нестабильные релизы: несогласованный mix источников, неконсистентный формат образцов и регрессии вида JSON-ответов вместо чистого текста.

## What Changes
- Добавляется нормативный профиль `1C-Expert-v4` с целевым размером датасета и базовыми pre-release порогами.
- Фиксируется обязательный mix источников для релиза:
  - `50%` реальный код 1C (BSL);
  - `30%` общий coding-контур;
  - `20%` русский instruction/chat контур.
- Фиксируется правило гранулярности 1C-корпуса: одна процедура/функция = один обучающий sample.
- Фиксируется контракт сериализации в plain text для RWKV:
  - `Instruction:` + `Response:`;
  - завершающий разделитель `<|endoftext|>`;
  - запрет на сырой JSONL-объектный формат в финальном train-text.
- Фиксируется обязательное перемешивание (shuffle/interleave) перед токенизацией.
- Фиксируется baseline-набор внешних референсных датасетов для coding и RU identity контуров.

## Impact
- Affected specs: `dataset-development`.
- Affected code:
  - `scripts/prepare_binidx.sh` (входной контракт `text` и токенизация готового текста);
  - новые/обновлённые скрипты curating/formatting/shuffle (в `scripts/` или `tools/`);
  - документация по датасетам (`README.md`, `docs/*` по data lifecycle).
